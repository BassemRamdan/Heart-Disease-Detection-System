1. Objectives

The primary goal of this project is to implement and evaluate a hybrid intelligent system for detecting heart disease risk, combining human-driven knowledge engineering with data-driven predictive modeling. The comprehensive tasks involve:

  • Dataset Processing & Visualization: Loading, cleaning, analyzing, and selecting features from clinical heart disease data.
  • Rule-Based Expert System: Designing a medical inference engine using 12 specialized rules.
  • Machine Learning: Training, tuning, and evaluating a Decision Tree Classifier.
  • Interactive Bonus UI: Deploying the entire hybrid engine on a responsive Streamlit Dashboard application.

2. Data Preprocessing & Exploratory Data Analysis

2.1 Data Processing & Normalization

The dataset was loaded and pre-processed thoroughly using Pandas and Scikit-Learn utilities. Missing values in clinical columns (such as `oldpeak` and `restecg`) were handled using Mean and Mode imputation. We normalized the numerical features (age, blood pressure, cholesterol, max heart rate) utilizing a `StandardScaler`. Categorical variables like chest pain (`cp`) and thalassemia (`thal`) were transformed using One-Hot encoding to adapt them for ML.

2.2 Visualization & Feature Selection

An interactive data analysis phase was executed utilizing Seaborn and Plotly. We derived a correlation heatmap of all clinical features against the target variable. Feature importance thresholds were applied (Threshold > 0.05), which successfully reduced our dataset dimensionality down to the 21 highest correlation features, significantly boosting ML stability.

3. The Rule-Based Expert System

3.1 Knowledge Base Definition

We engineered a pure Python inference engine designed to mimic a cardiologist's diagnostic thought process. We encoded 12 domain-specific medical rules to map patient vitals to risk levels:

Rule 1: If Cholesterol > 240 AND Age > 50 -> High Risk Rule 2: If Resting ECG = Normal AND Exang = No -> Low Risk Rule 3: If Thalassemia Defect (thal) in [1,2] -> Moderate Risk Rule 4: If Major Vessels Colored (ca) >= 2 -> High Risk...

3.2 Inference Performance

When evaluated mechanically over the dataset, the Expert System's strict adherence to medical criteria produced highly preventative flags (highlighting future risks regardless of current blockage), resulting in an exact Accuracy of 55.08% against the dataset's strict "current blockage" target definition.

4. The Decision Tree Machine Learning Model

4.1 Model Training

We utilized Scikit-Learn to build a DecisionTreeClassifier. The dataset was evaluated thoroughly, learning from complex non-linear combinations of the selected features (such as combining `Thalassemia` correlations with `ST Depression`).

4.2 Test & Evaluation Metrics

Upon evaluation, the Decision Tree model achieved remarkable precision and recall, dramatically outperforming the rigid Expert System in identifying complex underlying cardiovascular abnormalities.

  • Accuracy: 80.33%
  • Precision: 80.74%
  • Recall: 80.60%
  • F1-Score: 80.32%

5. Comparative Analysis & Remarks

System Architecture Evaluation Strategy Accuracy F1-Score
Rule-Based Expert System Preventative Medical Guidelines 55.08% 69.76%
Decision Tree ML Model Data-Driven Statistical Diagnosis 80.33% 80.32%

5.1 Explainability Analysis

The Expert System provides immense Explainability. It exactly matches and outputs sentences justifying its diagnosis (e.g., "Patient flagged due to: High BP (>140) with exercise-induced angina"). In contrast, the Decision Tree Model excels at Accuracy, efficiently inferring latent patterns that rigid human rules miss (e.g., classifying a young patient as High Risk due to a complex combination of Thalassemia and ST Depression scores), but behaves more like a "Black Box".

6. Streamlit User Interface (Bonus Feature)

To encapsulate the project, we engineered a flawless, interactive, and clinically-styled web interface using Streamlit. The system acts as a medical dashboard, enabling end-users to:

  • Input their vitals via responsive sliders and dropdown menus mapping precisely to the trained feature space.
  • Instantly view a dynamic "Radar Profile" graphing their heart metrics against average baselines.
  • Receive Hybrid Assessments: Both the ML prediction and the Expert System logic are fired in real-time, giving the physician a dual-perspective on the patient’s health!

7. Conclusion

This project elegantly showcases the strengths and weaknesses of different AI paradigms. While pure logic programming via an Expert System allows for absolute auditability in a medical context, Data-Driven Machine Learning drastically raises diagnostic accuracy by leveraging statistical realities. Combining both paradigms into a cohesive, user-friendly system provides a powerful demonstration of applied Intelligent Programming.
